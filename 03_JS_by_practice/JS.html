JS Basics

Javascript is a programming language that works with HTML content but is also a scripting language.
It is a dynamic typing high level language: Hight level programming language are easier to
read and write which allow js to recognize the value kind (type) on its own without having a human
to explicitly specify

Variables

Variables is a way to store / save a value, an expression or an objects (object literal or array) Storing values allows to save a value in order to re-use / rely on them for later logic implementation or just to save the value.
<variable-keyword> <variable-name> = <value>

There are 3 ways to implement them using different keywords

  • var <variable-name> : old/regular way that can be overridden too easily which can leads to unexpected behavior
  • let <variable-name> : variable for anticipated cases where it is supposed to be overridden
  • const <variable-name> : variable for primitives that would not change

Comparisons

Comparisons outputs boolean s. The comparisons operator are similar to the mathematical ones.
It applies to numbers in general, can be applied to strings also
but those are not incompatible with other primitives (types).

  • == : double equality (with 2 following operators - to avoid as possible)
  • === : triple equality (with 3 following operators - always prefer triple equality over the double equality)
  • != is not equal (with 2 following operators - to avoid)
  • !== is not equal (with 3 following operators - to always prefer)
  • <= : inferior or equal
  • >= : superior or equal
  • < : inferior
  • > : superior

Control Flow

A program can behaves based on conditions instead of just executing the code line by line: this is control flow. The ability to control the code flow could be done in 3 ways but always evaluating a condition. A condition is made up of the following concept: <value> <comparison-operator> <value> The result of that comparison will condition the code.

Four ways to handle conditions

Condition kind and Syntax Note

if [ [else if] else ] condition

if( <condition> ){ /*...*/ }
if( <condition> ) { /*...*/ }
else { /* ... */ }
if( <condition> ){ /* ... */ }
else if (<condition>) { /* ... */ }
else { /* ... */ }
  • efficient and flexible and common control flow
  • can check by evaluating only one function with just a if condition statement
  • can check 2 to more conditions with if else if statements
  • can check 2 and one default conditions with if else if else ( else being considered as default) statements

switch condition

switch(<value>) {
  case <value-to-check>:
     /* ... */
  case <value-to-check>:
     /* ... */
  default:
     /* ... */
}
  • useful when one want to check the value equality and describe different code case scenario

ternary condition

<condition>
 ? <what-to-do-if-true>
 : <what-to-do-if-false>
  • handy for quick checks
  • handy to conditionally assign a value for instance

Logical Operators

Logical operators are great to combined with condition and control flow: supporting, but also for assigning a value to a variable either:

  • possible values in one condition ("or" operator) - returning the first valid value
  • or one condition including multiple values to be checked ("and" operator) returning the first truthy value - returning the left-side value
  • or one condition supporting null or undefined as a value to be assigned (nullish coalescing operator) - returning the non-null or defined value
Operator Examples

|| "or" operator

<value> || <value>

&& "and" operator meant check if all the values are valid to compute a boolean - asserting all values are valid ( = truthy )

<value> && <value>

To evaluate in a condition

if( true && true ){
   // code will be executed

Assigning a value to a variable

?? nullish coalescing operator meant to output the the valid value not being null or undefined

<value> ?? <value>

    Functions

    Functions concept and types

    Functions allows to encapsulate a code block in order to be reused in another code implementation.
    It is characterized by () either prepended by a name or by the function keyword,
    followed by curly braces {} or =>.

    • one can create a function - named or not named
    • one can execute a declared or imported function - will always be named
    • one can assign a function to a variable - named or not named/ will inherit the name of the variable

    6 types of functions

    • function declaration / function definition:
      Creates/defines a function with a set of instruction and programming it to
      receive or not parameters or to return a value.
      function getName(name){return name || 'Stranger';}


    • function expression:
      Function declared with the function keyword and stored in a variable
      - this will inherits the variable name.
      const getName = function(name){return name || 'Stranger';}


    • anonymous function:
      Function declared without a name: generally to be used once.
      function(){return 'Doing something';}


    • IIFE or Immediate Invoked Function expression:
      Encapsulating the function itself and executing itself.
      ( function(){return 'Doing something';} )()


    • function constructor:
      Function created to return a new object - this should be executed with
      prepended new keyword as it creates a blueprint to
      be used to create new objects.
      // creating a function constructor
      function Person(name, age){
        this.name = name;
        this.age = age;
      }
      // creating a new object
      const person = new Person('Merlin', 140) ;


    • arrow function:
      Anonymous function assigned to a variable represented by
      parentheses only followed by =>
      ( <argument> ) => { /* ... */}


    Function parameters / arguments

    Like we've seen above, we can find sometimes passed element between parentheses:
    those are either parameters or arguments.
    Both are essentially doing the same - there are just called
    differently based on their context of use.

    Parameters or Arguments

    Both are representing data or any values to pass to the function which
    will use them in the function body = code logic.

    • parameters: this is what you pass in a function definition
      context (during the function creation)
    • arguments: this is what you pass in a function execution
      context (while calling the function)

    Function and returns

    As mentioned, functions are encapsulating a block of code that are re-used
    wherever the function is used and can return something or not.

    • when returning nothing, the code is applied and will by default return undefined
    • when returning something, the code is applied and output the result of the computation.
    • return is designed to exit the function (the function computed could be interrupted if wanted)

    Function and returns

    To return a value from the function definition, you must use the special keyword return
    followed by what you want to return once the code did run.

    • function not returning
      /* Function not explicitly returning anything
      - by default returns `undefined`*/

      function sayHi(){
        console.log("Hello, how are you?");
      }// undefined

    • function returning
      /* Function returning the name */

      function getSayHi(){
        return "Hello, how are you?";
      }// "Hello, how are you?"

    Data Structure (arrays and objects)

    DataStructure stores values in a useful way - Those corresponds to non-primitives

    • array: can store a list of things in a kind of ordered way: ex searching for the first item
    • object: can help to retrieve a known element such as words in dictionaries to get the value

    Both 2, open the doors to data structure by not only
    storing one array or object, but by expending the possibilities
    as we can nest those: example having an object within an object, or having an
    array within another array and this as much as needed.

    Array

    An array can be assimilated to a list of values;
    to each value it is attached a number that help accessing it
    (you could say you want the third item in the list).
    This is called a numeric index which does not start with 1 but with 0

    Arrays values can be accessed using the index of the item needed


    const list = [
      "one",
      "two",
      "three",
      "four",
      "five"
    ];

    // getting the first item using index
    const first = list[ 0 ];


    // getting the third item using index
    const third = list[ 2 ];


    // getting the third item using index
    const fifth = list[ 4 ];

    Modifying an array is possible: such as adding, modifying or delete an item from it.
    This can be done using the Object prototype methods ( functions belonging to the Array objects.)

    Object

    An object can be assimilated to a dictionary where one retrieves values by
    knowing the name as to each value is defined a name to access it.
    The name used to access a value is called a property

    Objects literals' values can be accessed

    • dot notation
    • (squared) bracket notation and the name of the property holding the value

    const person = {
      fullName: "Jane Doe",
      age: 0,
      profession: "unknown",
    };

    // getting the `fullName` dot notation
    const name = person.fullName;


    // getting the `fullName` using squared bracket
    const name = person[fullName];

    Modifying an object is possible: such as adding, modifying or delete a property and/or its value.
    This can be done using the Object prototype methods ( functions belonging to the Array objects.)

    Terminology review
    • assign: affecting a value to something
      assigning to a variable the value of 5
      var number = 5;

    • function declaration: defining a function with the function keyword
      creates/declares/defines a function "sayHello"
      function sayHello(){ /* ... */}

    • function expression: defining a function assigned to a variable.
      The function can be named or anonymous
      stores/assigns a function "sayHi" to a variable
      var newFunction = function sayHi(){ /* ... */}

    Loops

    Loops are a way to repeat a code/an action multiple times.
    4 types of loops are

    • for loop:
      allows to loop over a number of defined times or on Arrays
      useful to go over each item, or specific item, and do something with it
      It must instruct how to go through the loop in a statement between parentheses
      Syntax ex: for (var i = 0; i < 5 ; i++){/* ... */}

      Syntax ex: for (var i = 5; i > 5 ; i--){/* ... */}
    • for of loop:
      allows to loop over an entire Array
      useful to go over each item, or specific item, and do something with it
      beside stating the variable and the array - not accessing the index
      Syntax: for( var item of list){/* ... */}
    • for in loop:
      Allows to loop on Objects and Arrays.
      Useful to go over each property name of an Object or item of an Array.
      Syntax: for( var item in list){/* ... */}

      Example:

      var list = ["Banana", "Kiwi", "Mango"];
      var person = { firstName: "John", lastName: "Smith"};

      //looping over an `array`
      for( var item in list ){ console.log( item )};
      //"Banana"
      //"Kiwi"
      //"Mango"

      //looping over an `object`
      for( var property in person ){ console.log( property )};
      //"firstName"
      //"lastName"
    • while loop:
      Allows to loop (alternatively do something) continuously until a condition is met.
      Careful though, if the condition is never met, this can cause infinite loop.
      Syntax: while(<condition>){/* ... */}
      Example:

      var value = 0;
      while( value < 5 ){
        value++
      }
      console.log(value)//5
    • while loop:
      Allows to apply some code right before and while a condition is met.
      Careful though, if the condition is never met, this can cause infinite loop.
      Syntax: do {/* ... */ } while(<condition>);
      Example:

      var count = 0;
      do {
        count++;
      } while( value < 5 );
      // the count incrementation will occurs before checking the condition
      console.log(value)//5
    • forEach:
      Allows to loop over an Array.
      Careful though, if the condition is never met, this can cause infinite loop.
      Syntax: do {/* ... */ } while(<condition>);
      Example:

      var list = ["Banana", "Kiwi", "Mango"];
      list.forEach(function( item, i, arr ){
        /* ... */
      });

      list.forEach(function( item, i, arr ){
        /* ... */
      });
